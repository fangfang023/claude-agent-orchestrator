# 技术方案生成 Skill

## 概述

本 Skill 提供结构化的技术方案生成工作流程，能够将技术创意转化为结构清晰、逻辑严谨、语言专业的标准技术文档。

## 何时使用

**触发条件：**
- 用户要求生成"技术方案"、"技术实现方案"、"技术架构设计"
- 用户需要将创意转化为可工程实现的技术方案
- 用户需要详细的技术路线图和实施细节

## 文件结构

```
tech-solution/
├── README.md          # 本文件
├── GENERATE.md        # 生成指导
└── REVIEW.md          # 评审标准
```

## 使用方式

### 生成技术方案

```
用户：基于这个创意生成技术方案
→ 调用 tech-solution/GENERATE.md
→ 按照四个模块生成标准技术文档
```

### 审查技术方案

```
用户：检查这个技术方案的质量
→ 调用 tech-solution/REVIEW.md
→ 按照质量标准进行审查
```

## 输出规范

- **格式**：Markdown (.md)
- **命名**：`技术方案_[项目名称]_[日期].md`
- **编码**：UTF-8
- **输出位置**：`./generated_docs/`

## 特点

- ✅ 结构清晰：四个标准模块
- ✅ 逻辑严谨：从概述到实施的完整流程
- ✅ 语言专业：避免口语化，使用准确术语
- ✅ 可实施性强：包含详细的操作步骤和问题解决

## 依赖关系

技术方案通常作为其他文档的基础：
```
创意 → 技术方案 → 技术交底书
                → 工程技术论文
                → 商业价值分析
```
