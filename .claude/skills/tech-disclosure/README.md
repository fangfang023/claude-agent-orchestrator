# 技术交底书生成 Skill

## 概述

本 Skill 提供结构化的技术交底书生成工作流程，能够将技术创新转化为符合中国专利申请标准的专业技术交底书文档。

## 何时使用

**触发条件：**
- 用户要求生成"技术交底书"、"发明披露书"、"tech disclosure"
- 用户需要将技术创新转化为专利申请文档
- 用户需要记录技术发明以进行知识产权保护
- 用户完成技术项目需要准备专利申请材料

## 文件结构

```
tech-disclosure/
├── README.md          # 本文件
├── SKILL.md           # 技能定义（主要工作流）
├── GENERATE.md        # 生成指导（技术方案生成）
└── REVIEW.md          # 评审标准
```

## 使用方式

### 生成技术交底书

```
用户：基于这个创意生成技术交底书
→ 调用 tech-disclosure/SKILL.md
→ 按照八章节结构生成标准技术交底书
```

### 生成技术方案

```
用户：生成技术实现方案
→ 调用 tech-disclosure/GENERATE.md
→ 生成技术方案文档
```

## 输出规范

- **格式**：Markdown (.md)
- **命名**：`技术交底书_[发明名称]_[日期].md`
- **编码**：UTF-8
- **输出位置**：`./generated_docs/`

## 技术交底书结构

技术交底书包含以下八个必需章节：

1. **发明名称** - 简明扼要的技术方案名称（25字以内）
2. **技术领域** - 说明发明所属的技术领域
3. **背景技术** - 现有技术方案及其缺陷
4. **发明内容** - 技术问题、技术方案、有益效果
5. **附图说明** - 附图的简要说明（如有）
6. **具体实施方式** - 详细实施步骤和参数
7. **权利要求书** - 独立权利要求和从属权利要求
8. **摘要** - 发明要点的简要说明（300字以内）

## 特点

- ✅ 结构完整：符合专利局要求的八章节结构
- ✅ 逻辑严谨：从背景到实施的完整技术描述
- ✅ 语言专业：使用准确的技术术语
- ✅ 可实施性强：包含详细的实施方式和参数

## 与其他技能的关系

```
创意 → 技术方案 (GENERATE.md) → 技术交底书 (SKILL.md) → 专利申请
```

- **GENERATE.md**：用于生成技术实现方案
- **SKILL.md**：用于生成符合专利申请标准的技术交底书
- **patent-writing**：更详细的专利撰写指导

## 依赖关系

技术交底书通常作为专利申请的基础文档：
```
创意 → 技术交底书 → 专利申请书 → 专利代理完善 → 提交专利局
```
