# 技术交底书生成工作流

本技能提供结构化的工作流程，指导用户创建符合中国专利申请标准的技术交底书。

## 何时使用此工作流

**触发条件：**
- 用户要求生成"技术交底书"、"发明披露书"、"tech disclosure"
- 用户需要将技术创新转化为专利申请文档
- 用户需要记录技术发明以进行知识产权保护
- 用户完成技术项目需要准备专利申请材料

## 工作流程

### 第一阶段：信息收集

**目标：** 充分理解技术创新内容。

向用户询问：

1. **发明名称**：发明的简明名称（25字以内）

2. **技术领域**：本发明属于哪个技术领域？

3. **背景技术**：
   - 现有技术方案是什么？
   - 现有方案存在的主要问题/缺陷？

4. **发明内容**：
   - 要解决什么技术问题？
   - 核心创新点有哪些？
   - 技术方案是什么？
   - 有益效果是什么？

5. **实施方式**：
   - 如何实施本发明？
   - 关键技术参数是什么？
   - 是否有具体实施例？

**退出条件：**
当收集到足够信息以理解：
- 发明解决什么问题
- 发明如何解决问题
- 与现有方案的区别
- 关键技术特征

### 第二阶段：文档生成

#### 技术交底书标准结构

```markdown
# 技术交底书

## 一、发明名称
[简明扼要的技术方案名称]

## 二、技术领域
本发明涉及[具体技术领域]技术领域，特别是涉及[更具体的技术方向]。

## 三、背景技术

### 3.1 现有技术概述
[描述现有技术方案]

### 3.2 现有技术的缺陷
[客观指出存在的主要技术问题]

## 四、发明内容

### 4.1 要解决的技术问题
本发明要解决的技术问题是：[具体技术问题描述]。

### 4.2 技术方案

为解决上述技术问题，本发明提供了一种[发明类型]，其特征在于：

**核心创新点1：[创新点名称]**
[详细描述技术原理、实现方式、技术特征]

**核心创新点2：[创新点名称]**
[详细描述技术原理、实现方式、技术特征]

### 4.3 有益效果

与现有技术相比，本发明具有以下有益效果：

**效果1：[效果名称]**
[具体描述]

**效果2：[效果名称]**
[具体描述]

## 五、附图说明
[如有附图，按顺序说明]

## 六、具体实施方式

下面结合附图对本发明的具体实施方式进行详细说明。

### 实施例1：[实施例名称]
[详细描述实施例，包括技术方案、参数、步骤、条件、预期效果]

### 实施例2：[实施例名称]
[如需多个实施例，继续描述]

## 七、权利要求书

**权利要求1：**
一种[发明主题]，其特征在于，包括：
[必要技术特征1]；
[必要技术特征2]；
[必要技术特征N]。

**权利要求2：**
根据权利要求1所述的[发明主题]，其特征在于：
[附加技术特征]。

## 八、摘要

本发明公开了一种[发明名称]，涉及[技术领域]。该方法包括：[技术方案要点]。本发明实现了[主要效果]，具有[应用前景]。
```

### 第三阶段：质量审查

#### 审查清单

**内容完整性：**
- [ ] 所有8个必需章节都存在
- [ ] 背景技术清楚指出了问题
- [ ] 技术方案充分公开
- [ ] 有益效果有据可依
- [ ] 实施方式细节足以重现
- [ ] 权利要求覆盖所有创新点
- [ ] 摘要在300字以内

**术语一致性：**
- [ ] 全文使用相同的技术术语
- [ ] 权利要求术语与说明书一致

**专利符合性：**
- [ ] 发明名称在25字以内
- [ ] 权利要求使用正确的两段式
- [ ] 独立权利要求包含所有必要特征

## ⚠️ 审核阶段（生成后必须执行）

**技术交底书生成完成后，必须执行以下审核步骤：**

---

### 循环评审机制说明

本技能采用**自动循环评审机制**，确保文档质量达到标准：
- **合格标准**：评分 ≥ 80 分
- **改进阈值**：评分 < 80 分时触发重新生成
- **最大次数**：最多 3 次生成机会（包括初始生成）
- **循环控制**：每次评审后根据评分决定是否继续

---

### 审核执行流程

#### 第一步：读取审核标准

使用 Read 工具读取以下文件：
```
../../knowledge/review-standards/tech-disclosure-review.md
```

#### 第二步：初始化循环变量

```
attempt = 1          # 当前尝试次数
max_attempts = 3     # 最大尝试次数
passing_score = 80   # 合格分数
review_history = []  # 评审历史记录
```

#### 第三步：执行质量评审循环

**DO WHILE** (attempt ≤ max_attempts):

1. **执行质量审核**
   - 结构完整性检查：确认8个必需章节都存在
   - 内容质量检查：逐项检查各章节内容质量
   - 技术准确性检查：检查技术描述和逻辑一致性
   - 完整性检查：评估信息充分性和可实施性

2. **计算综合评分**

3. **记录本次评审**

4. **判断是否需要重新生成**
   ```
   IF 总分 >= 80 THEN
     → 退出循环，输出审核报告
   ELSE IF attempt < max_attempts THEN
     → 生成针对性改进意见
     → 重新生成文档
     → attempt = attempt + 1
   ELSE
     → 达到最大次数，退出并输出最终报告
   END IF
   ```

**END WHILE**

#### 第四步：生成针对性改进意见

**当评分 < 80 分时，必须生成具体可行的改进意见：**

```markdown
### 需要改进的问题

#### 问题 1：[具体描述]
- **位置**：[章节名称]
- **严重程度**：高/中/低
- **改进建议**：
  1. [具体操作1]
  2. [具体操作2]
```

#### 第五步：输出审核报告

包含：综合评分、尝试次数、评审历史、最终结论

## 后续行动

生成和审核完成后，建议：
1. 技术评审：组织技术团队评审
2. 专利代理人审阅：与专业专利代理人合作
3. 尽快申请：遵循先申请原则
4. 申请前保密：避免公开披露
