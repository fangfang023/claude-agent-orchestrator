# 技术交底书生成工作流

本技能提供结构化的工作流程，指导用户创建符合中国专利申请标准的技术交底书。

## 何时使用此工作流

**触发条件：**
- 用户要求生成"技术交底书"、"发明披露书"、"tech disclosure"
- 用户需要将技术创新转化为专利申请文档
- 用户需要记录技术发明以进行知识产权保护
- 用户完成技术项目需要准备专利申请材料

---

## 核心工作流程（重构版）

### 🔴 第零阶段：强制技术深度分析（新增）

**在开始撰写任何文档内容之前，必须完成以下分析：**

#### 步骤1：技术原理深度剖析

**对输入的技术方案进行深度分析，回答以下问题：**

1. **核心技术原理是什么？**
   - 技术的物理/数学/计算机科学基础是什么？
   - 技术通过什么机制实现其功能？
   - 关键技术步骤之间的因果关系是什么？

2. **各组成部分的功能与实现方式是什么？**
   - 每个模块/组件负责什么功能？
   - 每个模块如何实现其功能？（算法/硬件/协议）
   - 模块之间如何交互？

3. **技术创新点体现在哪里？**
   - 与现有技术的本质区别是什么？
   - 创新是结构上的、算法上的、还是应用场景上的？
   - 为什么这个创新能解决现有技术的问题？

4. **技术实现的难点和解决方案是什么？**
   - 实现本技术面临什么技术挑战？
   - 如何克服这些挑战？
   - 关键技术参数的选择依据是什么？

**⚠️ 关键要求：**
- **禁止套用模板**：必须基于对技术的真实理解进行分析
- **禁止词语堆积**：每个技术术语必须有明确含义和作用
- **确保逻辑连贯**：前后描述必须自洽，不能出现矛盾

#### 步骤2：技术逻辑链条构建

**构建完整的技术实现逻辑链条：**

```
输入 → 处理步骤1 → 处理步骤2 → ... → 输出
  ↓        ↓           ↓              ↓
[数据]  [技术手段1] [技术手段2]    [效果]
```

**对每个环节，必须说明：**
- 这个环节处理什么？
- 用什么技术手段处理？
- 为什么用这个手段？
- 这个环节的输入是什么？输出是什么？
- 这个环节与前后环节的关系是什么？

**⚠️ 禁止行为：**
- ❌ 跳过技术实现细节，只描述功能
- ❌ 使用"智能处理"、"自动分析"等空洞描述
- ❌ 堆砌技术术语而不说明其作用

#### 步骤3：技术深度自查

**在进入文档撰写前，完成以下自查：**

- [ ] 我能用通俗语言解释这个技术的工作原理吗？
- [ ] 我能说明每个技术模块的作用和实现方式吗？
- [ ] 我能说明技术实现的完整流程吗？
- [ ] 我能说明技术创新点在哪里吗？
- [ ] 我能说明技术参数的选择依据吗？

**如果任何一项为"否"，必须先深化技术理解，然后才能开始撰写。**

---

### 第一阶段：信息收集

**目标：** 充分理解技术创新内容。

**如果用户输入信息不足，需要询问以下问题：**

1. **发明名称**：发明的简明名称（25字以内）

2. **技术领域**：本发明属于哪个技术领域？

3. **背景技术**：
   - 现有技术方案是什么？
   - 现有方案存在的主要问题/缺陷？

4. **发明内容**：
   - 要解决什么技术问题？
   - 核心创新点有哪些？
   - 技术方案是什么？（具体说明实现方式）
   - 有益效果是什么？

5. **实施方式**：
   - 如何实施本发明？
   - 关键技术参数是什么？
   - 是否有具体实施例？

**退出条件：**
当收集到足够信息以理解：
- 发明解决什么问题
- 发明如何解决问题（包括技术实现方式）
- 与现有方案的区别
- 关键技术特征及其实现

### 第二阶段：文档生成（深度要求）

#### 技术交底书标准结构

```markdown
# 技术交底书

## 一、发明名称
[简明扼要的技术方案名称，25字以内]

## 二、技术领域
本发明涉及[具体技术领域]技术领域，特别是涉及[更具体的技术方向]。

## 三、背景技术

### 3.1 现有技术概述
[详细描述现有技术方案，说明各技术的核心原理和实现方式]

### 3.2 现有技术的缺陷
[客观指出存在的主要技术问题及产生原因]

## 四、发明内容

### 4.1 要解决的技术问题
本发明要解决的技术问题是：[具体技术问题描述]。

### 4.2 技术方案

为解决上述技术问题，本发明提供了一种[发明类型]，其特征在于：

**🔴 核心创新点1：[创新点名称]**

功能定位：
- 该创新点在整体技术方案中的作用
- 该创新点要解决的具体技术问题
- 输入和输出

实现机制：
- 采用什么技术实现？（算法/硬件/协议）
- 技术原理是什么？
- 关键技术参数是什么？
- 为什么选择这个技术？

具体实现：
- 硬件实现：[具体配置]
- 软件实现：[开发环境/框架]
- 算法流程：[详细步骤]

**🔴 核心创新点2：[创新点名称]**

[同上格式]

### 4.3 有益效果

与现有技术相比，本发明具有以下有益效果：

**效果1：[效果名称]**
- 原因：因采用[具体技术特征]
- 对比：[量化对比数据]
- 技术依据：[技术解释]

**效果2：[效果名称]**
[同上格式]

## 五、附图说明
[如有附图，按顺序说明]

## 六、具体实施方式

下面结合附图对本发明的具体实施方式进行详细说明。

### 实施例1：[实施例名称]

**一、硬件配置**
- [具体设备配置，包括型号、参数]

**二、软件参数**
- 开发环境：[编程语言、框架版本]
- 算法配置：[算法名称、关键参数]
- 部署方式：[部署架构]

**三、实施步骤**
步骤1：[步骤名称]
- 输入：[具体数据/条件]
- 处理：[采用什么技术，如何处理]
- 输出：[处理结果]
- 与下一步的关系：[为什么需要这个步骤]

步骤2：[步骤名称]
[同上格式]

**四、预期效果**
本实施例通过[技术手段]，预计可实现：
- [定性效果1]
- [定性效果2]

### 实施例2：[实施例名称]
[如需多个实施例，继续描述]

## 七、权利要求书

**权利要求1：**
一种[发明主题]，其特征在于，包括：
[必要技术特征1，包含实现机制]；
[必要技术特征2，包含实现机制]；
[必要技术特征N，包含实现机制]。

**权利要求2：**
根据权利要求1所述的[发明主题]，其特征在于：
[附加技术特征，包含实现机制]。

## 八、摘要

本发明公开了一种[发明名称]，涉及[技术领域]。该方法包括：[技术方案要点及实现机制]。本发明实现了[主要效果]，具有[应用前景]。
```

### 第三阶段：质量检查（新增）

#### 质量检查清单

**内容完整性：**
- [ ] 所有8个必需章节都存在
- [ ] 背景技术清楚指出了问题
- [ ] 技术方案充分公开，包含实现机制
- [ ] 有益效果有据可依，有技术依据
- [ ] 实施方式细节足以重现
- [ ] 权利要求覆盖所有创新点
- [ ] 摘要在300字以内

**术语一致性：**
- [ ] 全文使用相同的技术术语
- [ ] 权利要求术语与说明书一致
- [ ] 前后描述没有矛盾

**专利符合性：**
- [ ] 发明名称在25字以内
- [ ] 权利要求使用正确的两段式
- [ ] 独立权利要求包含所有必要特征

**技术深度检查：**
- [ ] 每个技术模块都说明了功能定位、实现机制、具体实现
- [ ] 每个技术选择都有依据
- [ ] 技术实现流程有完整的逻辑链条
- [ ] 避免了空洞的功能性描述

**逻辑连贯性检查：**
- [ ] 没有出现"智能处理"、"自动分析"等空洞描述
- [ ] 没有堆砌技术术语而不说明其作用
- [ ] 因果关系清晰（采用某技术→实现某效果）
- [ ] 前后描述呼应，没有矛盾

**词语堆积检测：**
- [ ] 没有使用"智能"、"先进"、"高效"等形容词而不说明具体含义
- [ ] 没有堆砌技术术语（如"大数据、人工智能、云计算"）而不说明作用
- [ ] 没有重复相同内容以凑字数

## 后续行动

生成和审核完成后，建议：
1. 技术评审：组织技术团队评审
2. 专利代理人审阅：与专业专利代理人合作
3. 尽快申请：遵循先申请原则
4. 申请前保密：避免公开披露

---

## 注意事项

1. **技术深度优先**：宁可少写一个功能，也要确保写清楚的技术功能有深度
2. **逻辑连贯**：每个技术描述必须有前因后果，不能孤立存在
3. **避免空洞**：禁止使用"智能处理"等不说明实现方式的描述
4. **术语一致**：全文使用相同的技术术语
5. **引用正确**：从属权利要求的引用关系必须正确
6. **保护范围合理**：既不能过宽容易被驳回，也不能过窄保护不足
7. **避免虚构数据**：使用定性描述或预期性描述，避免具体未经验证的实验数据
