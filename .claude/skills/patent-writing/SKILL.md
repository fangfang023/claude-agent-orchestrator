# 专利技术交底书撰写工作流

本技能提供结构化的工作流程，指导用户创建符合中国专利申请标准的技术交底书。作为专业的专利撰写指导，确保所有必需章节完整且格式正确。

## 何时使用此工作流

**触发条件：**
- 用户提及专利撰写："写专利"、"专利交底书"、"技术交底书"、"patent disclosure"
- 用户提及发明文档："发明文档"、"申请专利"
- 用户需要记录技术创新以进行知识产权保护
- 用户完成技术项目需要申请专利

**初始说明：**
解释本工作流程通过以下阶段指导创建完整的技术交底书：

1. **背景收集**：了解发明、技术领域和创新点
2. **文档结构**：按照专利标准构建必需章节
3. **内容撰写**：用适当的技术细节和术语撰写各章节
4. **质量审查**：验证完整性、一致性和符合专利要求

询问用户是否使用此结构化工作流或自由方式工作。

## 第一阶段：背景收集

**目标：** 充分理解发明，以撰写准确完整的交底书。

### 初始问题

向用户询问其发明：

1. **发明名称**：发明叫什么？（保持在25个汉字以内，使用标准技术术语）

2. **技术领域**：本发明属于哪个技术领域？（应匹配国际专利分类）

3. **背景技术**：
   - 该领域的现有技术方案是什么？
   - 现有方案存在的主要问题/缺陷是什么？
   - 是否有具体的现有技术文献可以引用？

4. **发明内容**：
   - 本发明要解决什么技术问题？
   - 核心创新点有哪些？
   - 技术方案/技术特征是什么？
   - 与现有技术相比的有益效果？

5. **实施方式**：
   - 发明如何实施？
   - 是否有具体实施例？
   - 关键技术参数是什么？

6. **附图**：是否有流程图、架构图或技术图？

**额外收集的上下文：**
- 相关项目文档或技术规范
- 架构图或系统设计
- 实验数据或测试结果
- 与现有方案的对比

**退出条件：**
当收集到足够的技术细节以理解：
- 发明解决什么问题
- 发明如何解决问题（技术方法）
- 与现有方案的区别
- 关键技术特征和参数

## 第二阶段：文档结构

**目标：** 创建包含所有必需章节的文档结构。

### 技术交底书必需章节

完整的技术交底书必须包含：

1. **发明名称**
2. **技术领域**
3. **背景技术** - 包含现有问题
4. **发明内容**：
   - 要解决的技术问题
   - 技术方案
   - 有益效果
5. **附图说明** - 如适用
6. **具体实施方式**
7. **权利要求书**
8. **说明书摘要**

### 创建文档结构

创建包含所有章节标题和占位符文本的初始文档。

**使用artifacts时：** 创建包含结构的artifact
**使用文件时：** 创建适当命名的markdown文件（如：`技术交底书-发明名称.md`）

告知用户结构已创建，现在可以填写各章节。

### 章节撰写顺序

推荐顺序（从核心技术内容开始）：
1. 背景技术 → 建立上下文
2. 发明内容（技术问题和技术方案）→ 核心创新
3. 具体实施方式 → 技术细节
4. 权利要求书 → 法律保护范围
5. 说明书摘要 → 总结
6. 发明名称 → 最终确定
7. 技术领域 → 最终确定
8. 附图说明 → 最终确定

## 第三阶段：内容撰写

对每个章节，遵循以下撰写指南：

### 1. 发明名称

**要求：**
- 最多25个汉字
- 使用标准技术术语（不使用杜撰词）
- 体现发明的应用领域
- 应与技术领域和权利要求范围匹配
- 避免：化学式、商标、商业口号

**格式：**
```
## 发明名称
**[简明扼要的技术方案名称，包括用途和类型]**
```

**示例：**
- "一种基于多模态大模型的PPT图片智能编辑方法" ✓
- "一种图像处理方法" ✓
- "超级智能图像编辑器" ✗（商业口号）

### 2. 技术领域

**要求：**
- 说明发明所属的直接技术领域
- 尽可能匹配国际专利分类
- 应体现发明的主题和类型
- 不应包含区别技术特征

**格式：**
```
## 技术领域
本发明涉及[具体技术领域]技术领域，特别是涉及[更具体的技术方向]。
```

**示例：**
```
本发明涉及人工智能和图像处理技术领域，特别是涉及一种基于多模态大模型的图片智能编辑方法。
```

### 3. 背景技术

**要求：** 包含三部分

**第一部分：公知技术基础**
- 简要介绍理解本发明所需的相关公知技术

**第二部分：最接近的技术方案**
- 至少引用一篇最接近的现有技术
- 说明其目的、技术措施和效果
- 描述现有技术方案如何实施
- 清楚说明与本申请的区别

**第三部分：现有技术的缺陷**
- 客观指出存在的主要问题
- 必须是技术性缺陷（不是管理性或商业性）
- 必须是本发明能够解决的问题
- 避免使用诽谤性语言

**撰写要点：**
- 使用中性、客观的语言
- 关注技术问题，不是商业问题
- 将缺陷与本发明的创新点对应
- 如引用具体文档，提供出处

### 4. 发明内容

#### 要解决的技术问题

**要求：**
- 明确说明要解决的技术问题
- 使用正面、直接、客观的语句
- 具体明确（避免"提高质量"等笼统说法）
- 体现发明类型（产品/方法）
- 体现与背景技术的区别
- 不包含技术方案的具体内容

**格式：**
```
### 要解决的技术问题
本发明要解决的技术问题是：[具体技术问题描述]。
```

#### 技术方案

**要求：**
- 核心章节 - 描述为达到目的而采取的技术措施
- 必须包括所有必要技术特征
- 充分公开到使本领域技术人员能够实施
- 用语应与权利要求书相应或相同
- 应概括一个范围，不是仅一个具体例子
- 可描述附加技术特征（另起段）

**结构：**
```
### 技术方案

为解决上述技术问题，本发明提供了一种[发明类型]，包括：

**核心创新点1：[创新点名称]**

[详细描述技术原理、实现方式、技术特征...]

**核心创新点2：[创新点名称]**

[详细描述技术原理、实现方式、技术特征...]

...

**技术特征总结：**
[列出关键技术特征，用编号或分点方式]
```

**撰写要点：**
- 使用清晰、精确的技术语言
- 描述结构、组织、形状等及作用、原理
- 说明技术特征之间的关系
- 包含必要的参数/范围
- 可用理论或分析支持

#### 有益效果

**要求：**
- 指出与现有技术相比的优点和效果
- 应清楚、有根据
- 使用以下一种或多种方式：
  - 对结构特征或作用关系进行分析
  - 用理论说明
  - 用实验数据证明

**内容可包括：**
- 生产率、质量、精度、效率的提高
- 能耗、原材料、工序的节省
- 加工、操作、控制、使用的简便
- 环境污染的治理
- 有用性能的出现

**格式：**
```
### 有益效果

与现有技术相比，本发明具有以下有益效果：

**效果1：[效果名称]**
[具体描述+对比+依据]

**效果2：[效果名称]**
[具体描述+对比+依据]

...
```

### 5. 附图说明

**如有附图：**
```
## 附图说明

**图1：[图名]**
展示[图的内容说明]

**图2：[图名]**
展示[图的内容说明]

...
```

**要求：**
- 按顺序说明所有附图
- 说明图示内容
- 局部剖视图要单独编号和说明

### 6. 具体实施方式

**要求：**
- 最详细章节 - 使发明可再现
- 至少一个具体实施方式（建议3个以上以获得更宽保护）
- 详细程度：使本领域技术人员能够重现，不必付出创造性劳动
- 所有配方及参数通常是一个具体的点（不是范围）
- 包含所有必要条件：材料、设备、参数等

**内容指南：**

**对于产品发明：**
- 说明组成部分
- 该各组成部分的位置关系或连接关系
- 说明各个部分起的作用
- 并说明该产品如何使用
- 可动作的产品还应说明其动作过程

**对于方法发明：**
- 说明工艺方法、工艺流程和条件参数（时间、压力、温度、浓度）
- 提交流程图
- 说明各步骤之间的关系和作用
- 给出参数的上下限范围及具体实例

**格式：**
```
## 具体实施方式

下面结合附图对本发明的具体实施方式进行详细说明。

### 实施例1：[实施例名称]

[详细描述实施例，包括：
- 具体技术方案
- 技术参数
- 实施步骤
- 实施条件
- 预期效果]

### 实施例2：[实施例名称]

[如需多个实施例，继续描述...]

### 实施例3：[实施例名称]

[如需多个实施例，继续描述...]
```

**重要注意事项：**
- 全文术语统一
- 各构件标有与附图一致的标记
- 权利要求中出现概括性技术特征时，给出多个实施方式
- 已知技术特征简写即可
- 详细说明区别现有技术的必要技术特征

### 7. 权利要求书

**要求：**
- 最重要章节 - 限定保护范围
- 必须清楚、简要，以说明书为依据
- 独立权利要求采用两段式：前序部分 + 特征部分

**结构：**
```
## 权利要求书

**权利要求1（独立权利要求）：**
一种[发明主题]，其特征在于，包括：
[必要技术特征1]；
[必要技术特征2]；
...
[必要技术特征N]。

**权利要求2（从属权利要求）：**
根据权利要求1所述的[发明主题]，其特征在于：
[附加技术特征]。

**权利要求3（从属权利要求）：**
...
```

**独立权利要求格式（两段式）：**
- **前序部分**：发明主题名称 + 与最接近现有技术共有的必要技术特征
- **特征部分**：使用"其特征在于..." + 区别于现有技术的技术特征

**撰写要点：**
- 用词严谨，避免模糊词语
- 避免否定句式
- 术语统一
- 每个技术特征都应有技术意义
- 能用结构特征时避免纯功能限定
- 不要重复限定同一技术特征

**从属权利要求：**
- 必须引用在前的权利要求
- 必须是对所引用权利要求的进一步限定
- 保护范围应落在被引用权利要求的保护范围之内
- 主题名称应与所引用权利要求一致

**重要约束：**
- 独立权利要求不得缺少必要技术特征
- 技术特征不得重复限定
- 避免多项引用多项从属权利要求的非择一引用
- 权利要求中不得出现插图或"如图...所示"
- 附图标记应加括号

### 8. 说明书摘要

**要求：**
- 简单扼要，全文不超过300字
- 不分段
- 说明发明名称和所属技术领域
- 清楚反映要解决的技术问题、技术方案要点及主要用途
- 不得出现商业性宣传用语
- 起始句："本发明公开了..."

**格式：**
```
## 摘要

本发明公开了一种[发明名称]，涉及[技术领域]。该方法包括：[技术方案要点]。本发明实现了[主要效果]，具有[应用前景]。
```

**示例：**
```
本发明公开了一种基于多模态大模型的图片智能编辑方法，涉及人工智能技术领域。该方法包括：使用混合OCR识别技术对图片进行文字识别；使用文生图模型去除图片中的文字；将识别结果转换为可编辑元素。本发明实现了AI生成图片的可编辑化转换，识别准确率达95%以上，具有广泛的工业应用前景。
```

## 第四阶段：质量审查

**目标：** 验证完整性、一致性和符合性。

### 审查清单

请用户验证：

**1. 内容完整性：**
- [ ] 所有8个必需章节都存在
- [ ] 背景技术清楚指出了问题
- [ ] 技术方案充分公开
- [ ] 有益效果有据可依
- [ ] 实施方式细节足以重现
- [ ] 权利要求覆盖所有创新点
- [ ] 摘要在300字以内

**2. 术语一致性：**
- [ ] 全文使用相同的技术术语
- [ ] 没有同一事物多种叫法
- [ ] 权利要求术语与说明书一致
- [ ] 附图标记一致

**3. 技术准确性：**
- [ ] 所有技术描述准确
- [ ] 参数和范围正确指定
- [ ] 技术特征描述恰当
- [ ] 特征之间的关系清楚

**4. 专利符合性：**
- [ ] 发明名称在25字以内
- [ ] 技术领域匹配专利分类
- [ ] 权利要求使用正确的两段式
- [ ] 权利要求中无模糊语言
- [ ] 独立权利要求包含所有必要特征
- [ ] 权利要求有说明书支持
- [ ] 摘要格式正确

**5. 文档质量：**
- [ ] 结构清晰逻辑性强
- [ ] 使用专业技术语言
- [ ] 无商业宣传用语
- [ ] 附图引用正确（如有）
- [ ] 现有技术引用恰当（如有）

### 最终验证

用户确认所有项目后，进行最终审查：

1. 通读整个文档
2. 检查流畅性和一致性
3. 验证章节间无矛盾
4. 确保权利要求和说明书一致
5. 确认发明充分公开

**如发现问题：** 指出需要修订的具体章节并解释原因。

**文档通过质量审查：** 祝贺用户并提供最终建议：

- 尽快申请（先申请原则）
- 申请前保密
- 如可能与专业专利代理人合作
- 准备附图（如专利局要求）
- 收集支持性实验数据

## 撰写技巧

### 核心原则

**1. 清晰精确：**
- 使用标准技术术语
- 避免模糊或歧义语言
- 必要时定义新术语
- 技术特征要具体明确

**2. 一致性：**
- 同一概念使用相同术语
- 保持格式一致
- 附图标记保持一致
- 权利要求与说明书对应

**3. 完整性：**
- 公开所有必要技术特征
- 提供足够的重现细节
- 适当时提供多个实施方式
- 覆盖所有创新点

**4. 专利策略：**
- 权利要求尽可能宽但仍具有新颖性
- 用详细说明书支持宽泛的权利要求
- 使用从属权利要求涵盖变体
- 适当时同时考虑产品和方法权利要求

### 常见错误

**1. 公开不充分：**
- 缺少关键技术特征
- 实施方式细节不足
- 参数范围模糊
- 缺少实验条件

**2. 术语问题：**
- 术语不一致
- 使用商业品牌名
- 使用杜撰的非技术词
- 描述模糊

**3. 权利要求问题：**
- 缺少必要特征
- 保护范围过窄
- 纯功能性权利要求
- 重复描述同一特征

**4. 结构问题：**
- 缺少必需章节
- 章节顺序不当
- 描述不完整
- 缺少具体例子

### 特殊考虑

**软件/计算机相关发明：**
- 必须使用技术手段解决技术问题
- 软件必须与硬件改进结合
- 描述软件算法和硬件架构
- 提供流程图和技术参数
- 包含具体实施示例

**方法发明：**
- 按顺序清晰描述工艺步骤
- 指定条件（时间、温度、压力等）
- 提供参数范围及具体示例
- 说明每个步骤的作用
- 复杂方法包含流程图

**产品发明：**
- 描述组件及其关系
- 解释组件如何相互作用
- 包含结构细节和材料
- 可动作产品描述操作
- 用图号标记组件

## 文件管理

**输出格式：**
- 主要格式：Markdown (.md)
- 考虑同时提供：Word (.docx) 或 PDF供专利代理人审阅
- 使用正确的UTF-8编码以支持中文字符

**文件命名规范：**
- 格式：`技术交底书-[发明名称].md`
- 使用简洁、描述性名称
- 文件名中避免特殊字符

**版本控制：**
- 撰写过程中跟踪迭代
- 保持重要版本的备份
- 多个修订时使用版本号

## ⚠️ 审核阶段（生成后必须执行）

**技术交底书撰写完成后，必须执行以下审核步骤：**

---

### 循环评审机制说明

本技能采用**自动循环评审机制**，确保文档质量达到标准：
- **合格标准**：评分 ≥ 80 分
- **改进阈值**：评分 < 80 分时触发重新生成
- **最大次数**：最多 3 次生成机会（包括初始生成）
- **循环控制**：每次评审后根据评分决定是否继续

---

### 审核执行流程

#### 第一步：读取审核标准

使用 Read 工具读取以下文件：
```
../../knowledge/review-standards/patent-writing-review.md
```

#### 第二步：初始化循环变量

```
attempt = 1          # 当前尝试次数
max_attempts = 3     # 最大尝试次数
passing_score = 80   # 合格分数
review_history = []  # 评审历史记录
```

#### 第三步：执行质量评审循环

**DO WHILE** (attempt ≤ max_attempts):

1. **执行质量审核**
   - 结构完整性检查：确认8个必需章节都存在
   - 内容质量检查：逐项检查各章节内容质量
   - 技术准确性检查：检查技术描述和逻辑一致性
   - 完整性检查：评估信息充分性和可实施性

2. **计算综合评分**

3. **记录本次评审**

4. **判断是否需要重新生成**
   ```
   IF 总分 >= 80 THEN
     → 退出循环，输出审核报告
   ELSE IF attempt < max_attempts THEN
     → 生成针对性改进意见
     → 重新生成文档
     → attempt = attempt + 1
   ELSE
     → 达到最大次数，退出并输出最终报告
   END IF
   ```

**END WHILE**

#### 第四步：生成针对性改进意见

**当评分 < 80 分时，必须生成具体可行的改进意见：**

```markdown
### 需要改进的问题

#### 问题 1：[具体描述]
- **位置**：[章节名称]
- **严重程度**：高/中/低
- **改进建议**：
  1. [具体操作1]
  2. [具体操作2]
```

#### 第五步：输出审核报告

包含：综合评分、尝试次数、评审历史、最终结论
